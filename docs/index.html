Everything You Need

A complete toolkit for managing your Caddy server, from basic site configuration to advanced monitoring and security.

Intuitive Dashboard

See all your sites at a glance. Add, edit, and manage reverse proxy configurations without touching a config file.

SSL Certificate Monitoring

Track certificate expiry dates with automatic notifications. Never let a certificate expire unexpectedly again.

Multi-User Support

Role-based access control with Admin, Editor, and Viewer roles. Two-factor authentication with TOTP support.

Docker Integration

See container status alongside your reverse proxy targets. Know instantly if a backend service is running.

Performance Monitoring

Track request rates, latency, error trends, and bandwidth per site. Prometheus metrics endpoint included.

Configuration History

Full rollback support with up to 50 versions. Made a mistake? Restore any previous configuration instantly.

Notifications

Get alerts via email, webhooks, or in-app notifications. Know immediately when certificates or domains are expiring.

Domain Management

Track domain registrations and renewal dates with WHOIS/RDAP lookups. Never miss a domain renewal.

Real-Time Logs

Tail Caddy logs in real-time with filtering by log level and domain. Debug issues without SSH access.

Get Started in Minutes

Choose your preferred installation method

Docker (Recommended)

The easiest way to run Caddyshack alongside your existing Caddy server:

docker run -d \
  --name caddyshack \
  -p 8080:8080 \
  -v /path/to/Caddyfile:/etc/caddy/Caddyfile \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  -e CADDYSHACK_AUTH_USER=admin \
  -e CADDYSHACK_AUTH_PASS=changeme \
  ghcr.io/djedi/caddyshack:latest

Mount your Caddyfile and optionally the Docker socket for container status integration.

Docker Compose

For a complete setup with Caddy and Caddyshack together:

version: '3.8'

services:
  caddy:
    image: caddy:latest
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - caddy_data:/data

  caddyshack:
    image: ghcr.io/djedi/caddyshack:latest
    ports:
      - "8080:8080"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile
      - /var/run/docker.sock:/var/run/docker.sock:ro
    environment:
      - CADDYSHACK_CADDY_API=http://caddy:2019
      - CADDYSHACK_AUTH_USER=admin
      - CADDYSHACK_AUTH_PASS=changeme

volumes:
  caddy_data:

Build from Source

For development or custom builds:

# Clone the repository
git clone https://github.com/djedi/Caddyshack.git
cd Caddyshack

# Build the binary
go build -o caddyshack ./cmd/caddyshack

# Run with environment variables
CADDYSHACK_CADDYFILE=/etc/caddy/Caddyfile \
CADDYSHACK_AUTH_USER=admin \
CADDYSHACK_AUTH_PASS=changeme \
./caddyshack

Configuration

Configure everything via environment variables

Variable Description Default
CADDYSHACK_PORT Port to listen on 8080
CADDYSHACK_CADDYFILE Path to Caddyfile to manage /etc/caddy/Caddyfile
CADDYSHACK_CADDY_API Caddy Admin API endpoint http://localhost:2019
CADDYSHACK_AUTH_USER Admin username admin
CADDYSHACK_AUTH_PASS Admin password —
CADDYSHACK_MULTI_USER Enable multi-user mode false
CADDYSHACK_DOCKER_ENABLED Enable Docker integration false
CADDYSHACK_METRICS_ENABLED Enable Prometheus metrics true

See the full documentation for all configuration options including email notifications, webhooks, and rate limiting.
